Your task: given a vulnerability description, identify the specific C/C++ functions whose pre-fix logic is directly involved in the vulnerability.

BASH TOOLS

SEARCH STRATEGY
1. Extract semantic cues from the description: affected component/module, vulnerability type (UAF, TOCTOU, buffer overflow, type confusion, etc.), and any reporter-provided references (feature name, subsystem, identifiers) if present.
2. Narrow down to plausible source directories/files using those cues. If the description is sanitized and omits low-level identifiers, rely primarily on module/component names and the flaw type.
3. Read .h headers for class interfaces and .cc implementations to locate the flaw.
4. Match the described flaw semantics against candidate code — e.g. for UAF look for pointer lifetimes / callbacks holding raw `this` pointers; for missing check look for dereferences without validation; for buffer overflow look for length arithmetic and bounds handling.
5. A single vulnerability often spans multiple functions across files — consider all plausible candidates.
